## ¿Por qué Astro?

La web no necesita más megabytes de JS solo para renderizar contenido estático.

Usamos **[Astro](https://astro.build)** como base para seguir entregando:

* **Cero JS por defecto**, ideal para landing pages y SEO
* **Arquitectura de islas**, habilitando interactividad selectiva
* **Tiempos de build rápidos** y optimizaciones out-of-the-box

Combinado con **React Islands** y **TailwindCSS**, Astro nos da el poder de entregar rendimiento en tiempo real *e* interfaces expresivas.
