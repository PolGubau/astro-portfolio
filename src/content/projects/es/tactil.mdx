## ¿Por qué Astro?

La web no necesita más megabytes de JS para renderizar contenido estático.

Usamos **[Astro](https://astro.build)** como base para seguir entregando:

* **Cero JS por defecto**, ideal para landing pages y SEO
* **Arquitectura de islas**, que permite interactividad selectiva
* **Tiempos de compilación rápidos** y optimizaciones incluidas

Combinado con **React Islands** y **TailwindCSS**, Astro nos da el poder de ofrecer rendimiento en tiempo real *y* interfaces expresivas.

## Stack tecnológico

| Herramienta / Librería | Propósito                               |
| ---------------------- | --------------------------------------- |
| `Astro`                | Generador de sitios estáticos           |
| `React`                | Islas de UI interactivas                |
| `TailwindCSS`          | Estilos de utilidad                     |
| `@astrojs/mdx`         | Contenido enriquecido con Markdown y JSX |
| `@astrojs/rss`         | Soporte RSS para blog/posts futuros     |
| `@astrojs/sitemap`     | Generación de sitemap optimizado para SEO |
| `@astrojs/react`       | Integración de React en Astro           |
| `@polgubau/animated`   | Mi propia librería de componentes de animación |
| `astro-meta-tags`      | Generación dinámica de meta tags        |
| `oxlint`               | Linter rápido para proyectos JS/TS modernos |
| `sharp`                | Procesamiento y optimización de imágenes |

## i18n y traducciones

El sitio web está construido con la internacionalización en mente. El contenido de texto se abstrae en archivos específicos por idioma (`en.ts`, `es.ts`, etc.).

Esto permite una expansión sencilla a otras regiones y admite la renderización dinámica de la UI por idioma.
